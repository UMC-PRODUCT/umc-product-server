## Domain Package 내부 구조

```
{domain}/
  ├── domain/       # 비즈니스 규칙
  ├── application/  # 유스케이스 + Port
  │   ├── port/
  │   │   ├── in/   # UseCase 인터페이스 - Command, Query 구분 (CQRS)
  │   │   └── out/  # Repository 인터페이스
  │   └── service/  # 구현
  └── adapter/      # 기술 구현
      ├── in/       # Controller, Scheduler
      └── out/      # Persistence, OAuth, S3
```

## Global vs. Common 패키지 구분 방법

| 구분    | global                           | common                 |
|-------|----------------------------------|------------------------|
| 관심사   | 기술적/인프라                          | 비즈니스/도메인               |
| 의존성   | Spring, JPA 등 프레임워크              | 비즈니스 개념                |
| 예시    | SecurityConfig, ExceptionHandler | BaseEntity, Address VO |
| 변경 이유 | 기술 스택 변경, 인프라 요구사항               | 비즈니스 요구사항              |

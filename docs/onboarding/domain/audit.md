# Audit Domain

## 역할

`audit` 도메인은 운영자가 중요한 변경 이력을 추적할 수 있도록 감사 로그를 저장하고 조회한다. 보안, 권한, 운영 책임을 확인하는 근거 데이터가 된다.

## 책임

- 감사 대상 이벤트를 구조화한다.
- 누가, 언제, 어떤 작업을 했는지 기록한다.
- 중앙운영사무국 권한을 가진 사용자에게 감사 로그 조회를 제공한다.

## 경계

감사 로그는 다른 도메인의 작업 결과를 기록하지만, 원본 도메인의 업무 규칙을 대신 판단하지 않는다. 각 도메인은 자신이 발생시킨 이벤트나 변경 사실을 감사 로그가 이해할 수 있는 형태로 전달한다.

## UX Writing Notes

감사 로그는 보안성이 높은 백오피스 표면이다. 문구는 짧게 쓰되 `감사 로그는 중앙운영사무국 국원만 조회할 수 있어요`처럼 접근 조건을 분명히 밝힌다.

# Maintenance Domain

## 역할

`maintenance` 도메인은 전체 서비스 또는 특정 도메인의 점검 일정을 관리하고, 점검 중인 요청을 우회하거나 차단한다.

## 책임

- 점검 일정을 생성, 수정, 종료한다.
- 점검 대상 도메인과 시간 범위를 검증한다.
- 점검 중인 요청에 점검 응답을 제공한다.
- 테스트/운영 우회 정책을 관리한다.

## 경계

점검 도메인은 다른 도메인의 기능을 직접 변경하지 않는다. 요청 필터와 정책을 통해 접근 가능 여부를 결정하고, 각 도메인의 비즈니스 규칙은 그대로 둔다.

## UX Writing Notes

점검 메시지는 사용자가 기다려야 하는 상황이므로 차분하게 쓴다. `서비스 점검 중이에요. 점검이 끝난 뒤 다시 시도해주세요`처럼 상태와 다음 행동을 함께 제공한다.

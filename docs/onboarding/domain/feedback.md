# Feedback Domain

## 역할

`feedback` 도메인은 사용자 피드백 템플릿과 응답을 관리한다. 특정 화면이나 기능에서 수집할 질문을 템플릿으로 제공하고, 사용자의 답변을 저장한다.

## 책임

- 피드백 템플릿을 조회한다.
- 피드백 대상과 컨텍스트를 구분한다.
- 사용자 답변을 템플릿 질문에 맞게 수집한다.

## 경계

피드백은 다른 도메인의 경험을 보완하기 위한 수집 도구다. 대상 리소스의 상태 변경이나 업무 규칙 판단은 원본 도메인이 담당한다.

## UX Writing Notes

피드백 문구는 사용자가 부담 없이 이해해야 한다. `템플릿`보다 화면에서는 `피드백 양식`처럼 목적을 설명하는 말을 우선한다.

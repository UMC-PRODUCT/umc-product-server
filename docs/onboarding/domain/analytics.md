# Analytics Domain

## 역할

`analytics` 도메인은 운영진 대시보드와 학교별 통계 화면에 필요한 지표를 조회한다. 출석, 상벌점, 스터디 그룹, 프로젝트 등 여러 도메인의 데이터를 읽어 운영자가 현재 운영 상태를 판단할 수 있게 만든다.

## 책임

- 운영진 대시보드 지표와 액션 큐를 제공한다.
- 학교 단위 요약 지표를 제공한다.
- 조회 기간, 정렬 조건, 접근 범위를 검증한다.
- 통계 조회 권한을 `ResourceType.ANALYTICS` 기준으로 제한한다.

## 경계

`analytics`는 지표를 만들기 위해 다른 도메인의 데이터를 읽을 수 있지만, 다른 도메인의 상태를 직접 변경하지 않는다. 지표 계산에 필요한 조회 로직은 서비스/포트에 두고, 컨트롤러는 요청 값을 전달하는 역할만 한다.

## UX Writing Notes

운영진이 보는 백오피스 화면이므로 권한 오류는 정확하게 쓴다. 정렬, 기간처럼 사용자가 바로 고칠 수 있는 문제는 `정렬 값을 확인해주세요`, `기간을 다시 선택해주세요`처럼 다음 행동을 말한다.

# Authentication Domain

## 역할

`authentication` 도메인은 로그인, OAuth 연결, JWT 인증, 이메일 인증, 비밀번호 정책을 다룬다. 사용자가 본인임을 증명하고 안전하게 세션을 유지하도록 돕는다.

## 책임

- OAuth 제공자 정보를 해석하고 연결 상태를 관리한다.
- JWT 인증 정보의 만료, 서명, 형식을 검증한다.
- 이메일 인증 세션을 생성하고 검증한다.
- 이메일/비밀번호 자격증명 정책을 검증한다.

## 경계

인증 도메인은 사용자를 식별하지만 회원 프로필 자체의 소유권은 `member` 도메인에 있다. 인증 성공 후 회원 정보가 필요하면 공개된 회원 UseCase를 통해 조회한다.

## UX Writing Notes

인증 실패 메시지는 보안을 위해 원인을 과도하게 노출하지 않는다. 사용자가 할 수 있는 행동인 `다시 로그인해주세요`, `인증을 다시 요청해주세요`, `이메일 주소를 확인해주세요`를 중심으로 쓴다.

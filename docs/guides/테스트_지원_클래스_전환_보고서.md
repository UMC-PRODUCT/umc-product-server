# UseCaseTestSupport 폐기(Deprecated) 및 IntegrationTestSupport 전환 보고서

## 1. 개요

본 문서는 `UseCaseTestSupport`를 Deprecated 처리하고, 신규 통합 테스트 기준 클래스를 `IntegrationTestSupport`로 통합한 배경과 변경사항을 정리한다.

## 2. 육하원칙(6W1H) 정리

| 구분             | 내용                                                                                                                                 |
|----------------|------------------------------------------------------------------------------------------------------------------------------------|
| **누가(Who)**    | UMC PRODUCT 백엔드 팀의 테스트 코드를 작성/유지보수하는 개발자                                                                                           |
| **언제(When)**   | `@Deprecated(since = "v2.0.0")` 적용 시점부터                                                                                            |
| **어디서(Where)** | `src/test/java/com/umc/product/support` 패키지 (`UseCaseTestSupport`, `IntegrationTestSupport`)                                       |
| **무엇을(What)**  | `UseCaseTestSupport`를 직접 사용하지 않도록 Deprecated 처리하고, 신규 통합 테스트의 공통 베이스를 `IntegrationTestSupport`로 통일                                 |
| **왜(Why)**     | 1) UseCase 전용처럼 보이는 이름과 실제 통합 테스트 구성 간 의미 불일치 해소, 2) API/UseCase 통합 테스트 기반 일원화, 3) 테스트 정책(트랜잭션/격리/모킹 범위) 명시 강화                     |
| **어떻게(How)**   | `UseCaseTestSupport`의 기존 설정/모킹 구성을 `IntegrationTestSupport`로 통합하고, `UseCaseTestSupport`는 `IntegrationTestSupport`를 상속하는 호환용 래퍼로 축소 |

## 3. 기존 방식과 변경 방식의 차이

| 항목        | 기존 방식 (`UseCaseTestSupport`)   | 변경 방식 (`IntegrationTestSupport`)                               |
|-----------|--------------------------------|----------------------------------------------------------------|
| 클래스 의미    | 이름상 UseCase 테스트 전용처럼 보임        | 통합 테스트 전반(API + UseCase) 공통 베이스를 명시                            |
| 구성 방식     | 클래스 자체에 어노테이션/외부 Mock 설정 직접 보유 | 공통 설정을 한 곳에 집중, 신규 테스트는 직접 상속 사용                               |
| 테스트 프로필   | 명시 없음                          | `@ActiveProfiles("test")` 명시                                   |
| 웹 테스트 지원  | 별도 지원 필드 없음                    | `@AutoConfigureMockMvc`, `MockMvc`, `ObjectMapper` 제공          |
| 영속성 검증 지원 | 별도 공통 필드 없음                    | `EntityManager` 공통 제공                                          |
| 정책 문서화    | 최소 수준                          | 트랜잭션 경계, DB 격리, 모킹 원칙, Fixture 사용 원칙까지 클래스 주석으로 명시             |
| 전환 전략     | 단일 베이스에 의존                     | 기존 테스트 호환을 위해 `UseCaseTestSupport`는 유지하되 Deprecated 처리 후 점진 이관 |

## 4. 실제 변경 항목

1. `UseCaseTestSupport`가 직접 설정을 가지는 구조에서, `IntegrationTestSupport`를 상속하는 구조로 변경되었다.
2. `UseCaseTestSupport`에 `@Deprecated(since = "v2.0.0")`가 추가되었고, 신규 테스트는 `IntegrationTestSupport` 직접 상속을 권장한다.
3. `IntegrationTestSupport`에 통합 테스트 공통 설정이 집약되었다.
    - `@SpringBootTest`, `@ActiveProfiles("test")`, `@Import(TestContainersConfig.class)`, `@Testcontainers`, `@DatabaseIsolation`, `@AutoConfigureMockMvc`
    - 외부 시스템 연동 Mock (`JavaMailSender`, `JwtTokenProvider`, `FirebaseMessaging`, `Storage`, `StoragePort`)
    - 공통 주입 필드 (`MockMvc`, `ObjectMapper`, `EntityManager`)
4. 현재는 하위 호환을 위해 기존 `UseCaseTestSupport` 상속 테스트가 함께 존재하며, 단계적으로 `IntegrationTestSupport` 직접 상속으로 이관하는 전환 단계다.

## 5. 기대 효과

- 테스트 베이스 클래스의 책임과 이름이 일치해 이해 비용이 감소한다.
- API/UseCase 통합 테스트가 동일 정책으로 동작해 테스트 일관성과 신뢰성이 높아진다.
- 설정 중복과 분산이 줄어 유지보수 시 변경 영향 범위를 예측하기 쉬워진다.

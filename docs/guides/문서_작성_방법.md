## 개요

`/docs` 디렉토리에 문서를 작성하는 방법을 설명합니다.

## 본문

### 파일 제목

한국어를 기반으로, 띄어쓰기 없이 `_`를 이용해 파일 이름을 작성합니다. 예: `docs_작성_방법.md`

### 문법

자유롭게 작성하되, 마크다운 형식을 적극적으로 활용하여 가독성을 높여주세요.

### Wiki 동기화

작성한 문서는 기본적으로 Wiki에도 포함하여 동기화해주세요.
본 디렉토리는 문서에 대한 형상관리를 소스코드와 함께 진행하기 위해 소스코드 내에 포함되기에, Wiki의 형상관리와는 별도로 진행됩니다.

### API ID / ErrorCode 자동 문서화

API ID와 ErrorCode 목록은 코드를 원본으로 자동 생성합니다.

```bash
./gradlew generateDocumentationCatalogs
```

생성 대상은 다음 파일입니다.

- `docs/guides/API_목록.md`
- `docs/guides/API_목록.json`
- `docs/guides/ErrorCode_목록.md`
- `docs/guides/ErrorCode_목록.json`
- `src/main/resources/static/docs/catalog/api/index.html`
- `src/main/resources/static/docs/catalog/api/catalog.md`
- `src/main/resources/static/docs/catalog/api/catalog.json`
- `src/main/resources/static/docs/catalog/error/index.html`
- `src/main/resources/static/docs/catalog/error/catalog.md`
- `src/main/resources/static/docs/catalog/error/catalog.json`

서버 실행 후 다음 경로에서 `markdown-it` 기반 정적 렌더링 문서를 확인할 수 있습니다.

- `/docs/catalog/api/`
- `/docs/catalog/error/`

새 API ID를 할당할 때는 prefix 기준으로 다음 번호를 확인합니다.

```bash
./gradlew nextApiId -Pprefix=CHALLENGER
```

새 ErrorCode를 할당할 때도 동일하게 prefix 기준으로 다음 번호를 확인합니다.

```bash
./gradlew nextErrorCode -Pprefix=CHALLENGER
```

자동 문서가 최신 상태인지 확인하려면 다음 명령을 사용합니다.

```bash
./gradlew checkDocumentationCatalogs
```

누락되거나 중복된 API ID와 ErrorCode를 확인하려면 다음 명령을 사용합니다. 기본 모드는 보고만 하고, CI에서 실패 처리가 필요하면 `-PstrictDocumentationCatalogs=true`를 함께 지정합니다.

```bash
./gradlew validateDocumentationCatalogs
./gradlew validateDocumentationCatalogs -PstrictDocumentationCatalogs=true
```
